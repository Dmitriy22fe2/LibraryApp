# Library Management System

Web-приложение для управления библиотечным фондом. Позволяет вести учет читателей, добавлять книги и закреплять их за читателями.
Проект реализован на базе Spring MVC с доступом к базе данных через JDBC Template


## Технологический стек

- Java 25
- Spring MVC
- JDBC Template
- PostgreSQL
- Docker Compose
- Maven
- Thymeleaf


## Конфигурация базы данных

Проект настроен на запуск "из коробки". Файл конфигурации `src/main/resources/database.properties` содержит дефолтные параметры подключения к локальной базе данных в Docker и функцию автоматической генерации таблиц с помощью бина `DataSourceInitializer` и SQL кода, сохраненного в `src/main/resources/schema.sql`.


## Инструкция по запуску

После открытия проекта в IDE, для того чтобы убедиться в правильности настройки путей, перезагрузите Maven. В IntelliJ IDEA откройте вкладку Maven и нажмите Reload All Maven Projects.

Разверните контейнер PostgreSQL с помощью IDE или выполнив команду в терминале:
```
docker-compose up -d
```
База данных поднимется на порту 5490. После выполнения команды подождите 5-10 секунд. СУБД требуется время на внутреннюю инициализацию, прежде чем она начнет принимать подключения.

Запуск приложения в IDE (IntelliJ IDEA). Проект построен на чистом Spring MVC (без Spring Boot), для запуска используется плагин SmartTomcat в IntelliJ IDEA:

1. Установите плагин SmartTomcat через настройки среды разработки.
2. Добавьте новую конфигурацию запуска (Run/Debug Configurations -> SmartTomcat).
3. В поле Tomcat Server укажите путь к папке со скачанным Apache Tomcat.
4. В поле Deployment directory выберите папку `src/main/webapp`.
5. В поле Context path оставьте только `/`.
6. Примените конфигурацию и запустите сервер Tomcat. Приложение будет доступно по адресу: http://localhost:8080/


## Навигация (Точки входа)

Основные разделы приложения:

- `/` Главная страница (содержит ссылки на ключевые разделы)
- `/people` Страница управления читателями. Функционал модуля: Просмотр списка читателей. Просмотр читателя и выданных ему книг. Добавление, редактирование и удаление читателей.
- `/books` Каталог книг. Функционал модуля: Просмотр списка книг. Просмотр книги. Добавление, редактирование, удаление, выдача и возврат книг.


## Особенности архитектуры

- Автогенерация таблиц: Схема базы данных создается автоматически при первом запуске приложения благодаря DataSourceInitializer.
- Целостность данных: На уровне сущностей настроена связь ON DELETE SET NULL. При удалении читателя из базы, закрепленные за ним книги не удаляются, а автоматически становятся свободными.
